# AI 어시스턴트 — 배경 & 설계

## 개요

꾹꾹이 안에 AI 어시스턴트(로드)를 탑재해서, 유저가 자신의 자산 데이터를 기반으로 질문하고 분석받을 수 있는 기능.

## 핵심 컨셉

- 유저가 "AI 질문하기" 화면에서 채팅 형태로 질문
- AI가 해당 유저의 자산 데이터(계좌, 종목, DCA, 수입/지출, 스냅샷 등)를 컨텍스트로 참조
- 로드 페르소나로 답변 (다정하고 조용한 말투, 문장 끝에 "냥")

## 예상 유스케이스

- "내 포트폴리오 분석해줘" → 보유종목/비중/수익률 기반 분석
- "이번 달 지출 줄일 수 있는 부분?" → 수입/지출 카테고리별 분석
- "DCA 한도 얼마나 남았어?" → 계좌별 한도/투자현황 답변
- "자산배분 리밸런싱 필요해?" → 목표 비중 vs 현재 비중 비교
- "올해 투자 수익률은?" → 스냅샷 데이터 기반 답변
- "다음 달 투자 계획 세워줘" → DCA 패턴 + 한도 잔여 기반 제안

## 기술 방향 (검토 필요)

### 백엔드
- Claude API (Anthropic) 연동
- 유저 질문 시 해당 유저의 자산 데이터를 조회 → system prompt에 컨텍스트로 주입
- 대화 히스토리 저장 (세션 또는 DB)

### 프론트엔드
- `/ai` 라우트 — 채팅 UI
- 메시지 입력 → 스트리밍 응답 표시
- Nav 자산관리 그룹 또는 별도 메뉴에 배치

### 데이터 컨텍스트 (유저별)
- 계좌 목록 + 보유종목 + 매수금액
- DCA 기록 (올해/전체)
- 자산배분 (목표 비중)
- 수입/지출 (최근 3개월)
- 주간 스냅샷 (최근 추이)
- 현금성 자산

### 보안 고려
- 유저 본인 데이터만 컨텍스트에 포함 (소유권 검증)
- API 키는 백엔드에서만 관리 (프론트 노출 금지)
- 요청 rate limit 필요
