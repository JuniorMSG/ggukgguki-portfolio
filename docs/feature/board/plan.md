# 게시판 — 구현 계획

## 구현 순서

1. DB 테이블 생성 (notice, board_request, board_comment, board_vote)
2. 백엔드 엔티티 + API (공지사항)
3. 백엔드 엔티티 + API (요청사항 + 댓글 + 투표)
4. 프론트 공지사항 페이지 (목록 + 상세)
5. 프론트 요청사항 페이지 (목록 + 상세 + 작성)
6. 프론트 댓글 + 좋아요/싫어요
7. 검색 + 필터
8. 관리자 기능 (상태변경, 고정글)

## 추후 구현 (필요 시)

- 대댓글 (parent_id 방식)
- 첨부파일/이미지 업로드 (S3 연동)
- 알림 (내 글에 댓글/상태변경 시 푸시)
- 태그 (기능별 태그, 다대다 관계)
- 북마크 (관심글 저장)
- 댓글 좋아요
- 마크다운 에디터
- 이메일 알림
- 통계 (요청 처리율, 평균 처리 시간)
- 관리자 대시보드 (미처리 요청 건수 등)
